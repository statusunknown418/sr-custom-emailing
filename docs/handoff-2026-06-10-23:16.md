## Mental model clarified (for the next agent)
- **Clay enriches LEADS only** and calls `/<flow>/generate` (per row, 1-element
  `leads` array, `x-internal-secret` header). It does NOT scrape posts.
- **Post scraping is triggered upstream** ‚Äî now via the Discord `our-posts` /
  `someone-else` commands (or the `/<flow>/scrape` endpoints).
- A post must be scraped+cached BEFORE its leads hit `/generate`, else generate
  fails loud (`posts not scraped yet: <url>`). Scrape is async (returns a runId;
  D1 write lands when the task finishes).
- Instantly push (Flow B): per-lead `POST /api/v2/leads` with `campaign`, `email`,
  `first_name`/`last_name`, `company_name`, and the 6 rendered email fields as
  `custom_variables` (campaign steps must reference `{{email1Body}}` etc.).
